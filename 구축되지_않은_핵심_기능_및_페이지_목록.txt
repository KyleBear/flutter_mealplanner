🗓️ 나만의 스케줄 캘린더 앱 v0.2 - 칼로리/영양성분 기반 추천 시스템
📋 구축되지 않은 핵심 기능 및 페이지 목록

================================================================================
🚫 구축되지 않은 핵심 페이지들 (완전 미구축)
================================================================================

1. 건강 프로필 설정 페이지 (신규 - 핵심)
   - 기본 신체 정보 입력 (키, 몸무게, 나이, 성별)
   - 활동 수준 설정 (좌업생활, 가벼운 운동, 활발한 운동 등)
   - 건강 목표 선택 (체중감량, 근육증가, 유지, 체중증가)
   - 알레르기/식이제한 설정 (비건, 글루텐프리, 견과류 알레르기 등)
   - 기초대사율(BMR) 자동 계산
   - 일일 권장 칼로리 자동 산출 

2. 영양 목표 관리 페이지 (신규 - 핵심)
   - 일일/주간/월간 칼로리 목표 설정
   - 3대 영양소 비율 설정 (탄수화물:단백질:지방)
   - 미량 영양소 목표 설정 (비타민, 미네랄)
   - 목표 달성률 시각화 (프로그레스 바, 차트)
   - 목표 수정 및 히스토리 관리
   - 기본 목표 추천 (일반적인 건강 가이드라인 기반)

3. 칼로리/영양 추천 엔진 페이지 (신규 - 핵심)
   - 단순 계산 기반 개인화 추천 알고리즘
   - BMR + 활동량 기반 칼로리 계산
   - 잔여 칼로리 기반 단순 추천
   - 부족한 영양소 우선 추천 (기본 영양학 공식)
   - 시간대별 맞춤 추천 (아침, 점심, 저녁, 간식)
   - 날씨/계절 고려 추천 (기본 규칙 기반)
   - 사용자 선호도 기반 추천 (단순 필터링)
   - 유사 사용자 기반 추천 (기본 통계 기반)

4. 개인 영양 대시보드 페이지 (신규)
   - 일일/주간/월간 영양 섭취 단순 계산 분석
   - 목표 대비 달성률 기본 시각화
   - 영양소별 기본 트렌드 분석 (단순 통계)
   - 기본 건강 점수 (단순 공식 기반)
   - 체중 변화 추적 그래프
   - 운동량과 칼로리 소모 기본 연동

5. 식품 데이터베이스 탐색 페이지 (신규)
   - 포괄적 식품 영양 정보 DB
   - 바코드 스캔으로 제품 정보 조회
   - 사용자 정의 레시피 등록
   - 영양성분 상세 분석
   - 대체 식품 추천
   - 즐겨찾기 및 최근 검색 기록

================================================================================
🗄️ 구축되지 않은 데이터 모델들
================================================================================

1. NutritionGoal Collection (신규)
   - goalId: string
   - userId: string
   - targetDate: Timestamp
   - dailyCalories: double
   - macroRatios: Map (탄수화물:단백질:지방 비율)
   - microNutrients: Map<string, double> (비타민, 미네랄)
   - isActive: bool
   - createdAt: Timestamp
   - updatedAt: Timestamp

2. FoodItem Collection (신규)
   - foodId: string
   - name: string
   - nameKor: string
   - category: string
   - nutritionPer100g: Map (100g당 영양성분)
   - allergens: List<string>
   - barcode: string (optional)
   - brand: string (optional)
   - imageUrl: string
   - source: string (USDA, FDA, user_input)
   - verificationStatus: string
   - createdAt: Timestamp

3. MealRecord Collection (신규)
   - recordId: string
   - userId: string
   - scheduleId: string (연결된 스케줄)
   - mealType: string (breakfast, lunch, dinner, snack)
   - consumedAt: Timestamp
   - foodItems: List<Map> (섭취한 음식 목록)
   - totalCalories: double
   - totalNutrition: Map (총 영양성분)
   - satisfactionRating: int (1-5)
   - notes: string
   - photos: List<string>
   - createdAt: Timestamp

4. NutritionAnalysis Collection (신규)
   - analysisId: string
   - userId: string
   - analysisDate: Timestamp
   - period: string (daily, weekly, monthly)
   - totalCaloriesConsumed: double
   - calorieGoalAchievementRate: double
   - macroBalance: Map (3대 영양소 균형)
   - microNutrientStatus: Map<string, Map> (미량영양소 상태)
   - healthScore: double
   - recommendations: List<string>
   - trends: Map (체중 변화, 에너지 레벨)
   - createdAt: Timestamp

5. RecommendationHistory Collection (신규)
   - recommendationId: string
   - userId: string
   - recommendationType: string (meal, snack, recipe)
   - recommendedAt: Timestamp
   - context: Map (추천 컨텍스트)
   - recommendedItems: List<Map> (추천된 항목들)
   - userAction: string (accepted, rejected, modified, ignored)
   - feedback: Map (사용자 피드백)
   - learningWeight: double

🎯 구축되지 않은 핵심 기능들
================================================================================

1. 건강 관리 시스템
   - 기초대사율(BMR) 자동 계산 (Harris-Benedict 공식)
   - 일일 권장 칼로리 자동 산출
   - 3대 영양소 비율 설정 및 관리
   - 미량 영양소 목표 설정
   - 체중 변화 추적 및 그래프

2. 계산 기반 추천 시스템
   - 실시간 계산 기반 개인화 추천
   - 시간대별 맞춤 추천
   - 잔여 칼로리 기반 추천
   - 부족한 영양소 우선 추천
   - 날씨/계절 고려 추천 (규칙 기반)
   - 사용자 선호도 기반 추천 (필터링)

3. 영양 분석 시스템
   - 일일/주간/월간 영양 섭취 분석
   - 목표 대비 달성률 시각화
   - 영양소별 트렌드 분석
   - 건강 점수 및 개선 제안
   - 영양 균형 점수

4. 외부 API 연동
   - USDA FoodData Central API (영양 정보)
   - OpenFoodFacts API (바코드 스캔)
   - Spoonacular API (레시피 정보)
   - Google Fit API / Apple HealthKit (운동량 추적)

📝 결론
================================================================================

현재 구축된 코드는 기획안의 10-15% 수준으로, 기본적인 사용자 인증과 식단 관리 기능만 구현되어 있습니다.

기획안의 핵심인 건강 관리, 영양 분석, 계산 기반 추천 시스템은 전혀 구축되지 않았으며, 이는 단순한 식단 관리 앱에서 개인화된 건강 관리 플랫폼으로 발전시키기 위해 추가 개발이 필요한 상태입니다.

특히 다음 기능들이 우선적으로 개발되어야 합니다:
1. 건강 프로필 설정 및 BMR 계산
2. 영양 목표 관리 시스템
3. 계산 기반 추천 엔진
4. 영양 분석 대시보드
5. 식품 데이터베이스 및 바코드 스캔

이러한 기능들이 구축되면 기획안의 90% 이상을 달성할 수 있을 것입니다.
